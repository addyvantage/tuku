# Tuku v1 Bootstrap Contracts

This repository is bootstrapped around these invariants:

1. Tuku owns canonical conversation and state.
2. Codex is an execution worker behind adapter contracts.
3. Canonical response rule is mandatory.
4. Layered state model is explicit in contracts.
5. Work Capsule is object-centric in v1 and graph-friendly for later evolution.
6. Local-first runtime and storage are the authority.
